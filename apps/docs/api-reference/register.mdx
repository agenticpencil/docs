Create a new AgenticPencil account programmatically. Perfect for onboarding users directly from your application or for automated account creation workflows.

<RequestExample>

```bash cURL
curl -X POST "https://api.agenticpencil.com/v1/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "SecurePassword123!",
    "name": "John Doe",
    "company": "Example Corp"
  }'
```

```python Python
import requests

url = "https://api.agenticpencil.com/v1/auth/register"
data = {
    "email": "user@example.com",
    "password": "SecurePassword123!",
    "name": "John Doe",
    "company": "Example Corp"
}

response = requests.post(url, json=data)
result = response.json()
```

```javascript Node.js
const axios = require('axios');

const response = await axios.post(
  'https://api.agenticpencil.com/v1/auth/register',
  {
    email: 'user@example.com',
    password: 'SecurePassword123!',
    name: 'John Doe',
    company: 'Example Corp'
  }
);
```

</RequestExample>

## Authentication

<Info>

## Request Body

<ParamField body="email" type="string" required>
  Valid email address for the new account
</ParamField>

<ParamField body="password" type="string" required>
  Password meeting security requirements (min 8 characters, 1 uppercase, 1 lowercase, 1 number)
</ParamField>

<ParamField body="name" type="string" required>
  Full name of the user
</ParamField>

<ParamField body="company" type="string">
  Company or organization name (optional)
</ParamField>

<ParamField body="referral_code" type="string">
  Referral code for bonus credits (optional)
</ParamField>

## Response

<ResponseField name="status" type="string">
  Status of the request ("success" or "error")
</ResponseField>

## Email Verification

<AccordionGroup>
  <Accordion title="When is verification required?">
    Email verification is required for:
    - New account registrations
    - Email addresses from certain domains (for security)
    - Accounts requesting higher initial limits
  </Accordion>

  <Accordion title="Verification process">
    1. User registers with this endpoint
    2. Verification email sent automatically
    3. User clicks verification link in email
    4. Account becomes fully active
    5. API key becomes available in dashboard
  </Accordion>

  <Accordion title="What if verification email doesn't arrive?">
    Users can:
    - Check spam/junk folders
    - Request a new verification email from the dashboard
    - Contact support if issues persist
  </Accordion>

  <Accordion title="API access before verification">
    - If `verification_required` is true: No API access until verified
    - If `verification_required` is false: Immediate API access with returned key
  </Accordion>
</AccordionGroup>

## Common Registration Errors

<AccordionGroup>
  <Accordion title="EMAIL_EXISTS - Email already registered">
    ```json
    {
      "status": "error",
      "error": "Email already registered",
      "message": "An account with this email address already exists",
      "code": "EMAIL_EXISTS"
    }
    ```
    **Solution**: Use a different email or try logging into the existing account
  </Accordion>

  <Accordion title="WEAK_PASSWORD - Password doesn't meet requirements">
    ```json
    {
      "status": "error",
      "error": "Password too weak",
      "message": "Password must be at least 8 characters with uppercase, lowercase, and number",
      "code": "WEAK_PASSWORD"
    }
    ```
    **Solution**: Create a stronger password meeting all requirements
  </Accordion>

  <Accordion title="INVALID_EMAIL - Email format invalid">
    ```json
    {
      "status": "error",
      "error": "Invalid email format",
      "message": "Please provide a valid email address",
      "code": "INVALID_EMAIL"
    }
    ```
    **Solution**: Check email format and ensure it's valid
  </Accordion>

  <Accordion title="RATE_LIMITED - Too many registration attempts">
    ```json
    {
      "status": "error",
      "error": "Too many registration attempts",
      "message": "Please wait before attempting another registration",
      "code": "RATE_LIMITED"
    }
    ```
    **Solution**: Wait 15 minutes before trying again
  </Accordion>
</AccordionGroup>

## Next Steps After Registration

<Steps>
  <Step title="Email Verification">
    If required, complete email verification through the link sent to the user's inbox
  </Step>
  <Step title="Dashboard Access">
    Visit [platform.agenticpencil.com](https://platform.agenticpencil.com) to access the full dashboard
  </Step>
  <Step title="API Key Management">
    Generate additional API keys or manage existing ones through the dashboard
  </Step>
  <Step title="First API Call">
    Test API access with a simple call to the usage endpoint
  </Step>
  <Step title="Plan Upgrade">
    Consider upgrading to Pro or Scale plans for higher limits and priority support
  </Step>
</Steps>
